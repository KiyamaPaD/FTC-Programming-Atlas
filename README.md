# FTC Programming Atlas

> Interactive node-based documentation website for FTC programming concepts, built as a visual learning atlas for future team generations.

## Live Demo

[View Live Demo](https://ftcprogrammingatlas.netlify.app)

## Overview

FTC Programming Atlas is a front-end web project designed to make FTC programming knowledge easier to navigate, understand, and pass on.

Instead of keeping notes in scattered files or long documents, this project turns documentation into an interactive map of nodes and relationships. Each node represents a real FTC programming topic, such as TeleOp, Autonomous, PID, Localization, Pathing, or Troubleshooting.

The main goal is to create a visual internal manual that helps future team generations learn faster and understand how important concepts connect to each other.

## Why I Built It

In robotics teams, useful technical knowledge often gets lost between seasons or remains spread across random files, chats, and unfinished documents.

I wanted to build a system that:
- feels more intuitive than classic documentation
- is easier to explore for new programmers
- connects concepts visually
- helps preserve knowledge inside the team
- can become a real teaching tool for future members

This project is both a practical team documentation tool and a portfolio project that reflects my interest in programming systems, UI design, structure, and education.

## Main Features

- Interactive node-based documentation map
- Full-screen topic viewer for detailed explanations
- Editable relations between concepts
- Drag and drop node positioning
- Automatic overlap prevention between nodes
- Search system for topics and content
- Zoom in, zoom out, fit view, and center selection
- Undo and redo history
- Intro screen and built-in tutorial
- Local browser persistence

## How It Works

Each topic is represented by a node on the map.

Users can:
- select a node
- open full documentation
- create new topics
- edit titles, categories, and content
- connect nodes through labeled relations
- reorganize the map visually

This makes the documentation feel closer to a concept graph than to a traditional static wiki.
